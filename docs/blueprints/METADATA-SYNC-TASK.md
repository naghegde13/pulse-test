### (M/K) AdvanceTimeDimension — re-tier to 2 user fields (ADR 0023)
Today (`V132`): **20 fields, all exposed as user-configurable.** Target: user-facing = **2**
(`target_scope`, `advance_to`); the other ~18 system-derived + **inspectable, not editable**.
**⚠️ Metadata-MODEL gap:** there is **no `derived`/`readonly` flag** in `params_schema` today to
express the user-vs-derived split. Representing it needs either a `params_schema` convention
(e.g. `tier: user|derived` per field) or a new column — **a small design decision for the
Builder-architecture session** (it's how the catalog expresses what the compiler must derive).

### (M/K) RemotePipelineInvocation — add `invocation_mode` (ADR 0021)
`V131` param schema lacks `invocation_mode` (async|sync) — add it. Broker refs
(`federated_tenant_key`, `remote_target_ref`, `airflow_connection_id`) become **derived** (same
2-tier question as above).
